# Product Positioning

## The Wedge

ClawShield should not present itself as another abstract AI firewall. The market is already crowded with guardrails, dashboards, policy engines, and hosted detectors.

The sharp position is narrower and stronger:

**ClawShield is the local-first trust layer that makes OpenClaw feel safe enough to run daily.**

## Core Promise

- It helps before the model sees risky content.
- It cleans what should not live forever in transcripts.
- It inspects skill bundles before trust quietly expands.
- It turns OpenClaw’s scattered security controls into one readable posture.

## Differentiators

### Local By Default

No account, no quota, no cloud dependency in the baseline path.

### Operator UX

The output must help a human decide quickly. Findings should not read like scanner noise. Each one needs:

- what happened
- why it matters
- what to do next

### Supply-Chain Awareness

Runtime enforcement matters, but so does the moment before installation. Skill inspection is a first-class feature, not a sidecar.

### Composition With OpenClaw

ClawShield should amplify:

- security audit
- tool policy
- exec approvals
- pairing
- plugin allowlists

It should not pretend these controls do not exist.

## Messaging Guardrails

- Never say prompt injection is solved.
- Never imply the product is a sandbox or kernel boundary.
- Never bury tradeoffs.
- Keep the tone calm and exact.
